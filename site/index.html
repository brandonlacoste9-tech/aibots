live · paper research only

aibots

Research brain for a paper-trading desk. Proposes, never submits orders. Forced market tools, one bull proposal, one bear critic — everything journaled for human confirm.

View on GitHub CLI docs

Control plane

forced tools → bull proposal → bear critic → journal → human confirms

Tools (always first)

  • get_price_history — yfinance + cache
  • compute_indicators — SMA / RSI / MACD / BB
  • get_news — Finnhub (optional key)

Proposals (record only)

  • propose_order — submitted: false
  • decide_hold — no trade, still journaled
  • Bear critic uses the same tool outputs

Bull proposal

Model may only propose after forced research. Thesis must cite tool numbers — not invented RSI/MACD.

Bear critique

One extra pass argues the strongest concrete bear case from the same data. Human sees both sides before confirm.

Quick start

pip install -e ".[dev]"
cp .env.example .env   # XAI_API_KEY, optional FINNHUB_API_KEY
python -m aibots research AAPL
python -m aibots journal --limit 5

This Netlify site is docs only. The research loop runs via CLI or an API host — not in the browser.
